docker 基础指令

发表时间 2017-01-03   |   阅读次数

基础概念

images:镜像,类型软件安装包

基础指令

container:容器，用于运行images

docker images 查看所有的镜像

docker images rmi [image id]

docker ps -[options] 查看运行或历史运行的container

docker ps -a 查看所有container

docker ps 当前运行的容器

docker start -[options] [image name] 根据image启动新的container

docker run -[options] [container name] 启动container

1.端口映射 -p [host port]:[container port]
2.挂载本地文件 -v [host file]:[container file path]

docker exec -[option] [container name] [command] 执行该容器下的命令

docker stop [container id] 停止容器

docker export [container id] > [container snapshot name] 导出容器

cat [container snapshot file path] | docker import - [REPOSITORY NAME]:[TAG NAME] 导入一个容器快照作为镜像，并指定名

docker rm [container id] 删除一个已终止的容器

docker rm -f [container id] 强制删除一个运行中的容器

image 构建

docker 构建image有两种方式:

1. 从docker.io pull images
2. 通过Dockerfile进行构建: 
命令为 docker build . 运行该指令后，会在当前路径下搜寻名为Dockerfile文件

Dockerfile 构建images的语法简介:

Dockerfile语法有两部分组成:

1. 注释 # comments
2. 命令+参数: [command] [argument]

command命令

FROM FROM命令必须是Dockerfile的首个命令。如FROM centos:tag

MAINTAINER 用于声明作者,一般放在FROM命令后面

RUN 运行shell指令

WORKDIR 切换工作路径

CMD 设置container启动时执行的操作 有三种命令形式:

1. CMD ["executable","param1","param2"] (like an exec, this is the preferred form)
2. CMD command param1 param2 (as a shell)
3. CMD ["param1","param2"] (as default parameters to ENTRYPOINT) 需使用ENTRYPOINT配合: ENTRYPOINT ["executable"]

ENTRYPOINT 有两种方式:

1. ENTRYPOINT ["executable", "param1", "param2"] (like an exec, the preferred form)
2. ENTRYPOINT command param1 param2 (as a shell)

tips:

1. CMD 和 ENTRYPOINT 指令，dockerfile 只执行最后一条

COPY image 中文件从一个路径copy到另一个路径

ADD 将本地文件拷贝到image中

USER 切换用户

ENV 设置环境变量如: ENV JAVA_HOME /usr/bin/java

EXPOSE 指定暴露端口

ARG 设置构建时变量，镜像建立完成后就失效

docker 网络

--link

将2个容器做单向连接
for example
docker run -d --name test1 redis 根据image redis 创建容器test1

docker run -d --name test2 --link test1 php 根据image php创建容器test2，并将test2 link 到 test1，使得在test2中可以通过容器名test1对其进行访问

docker network

为docker 容器创建一个网络，后续可以将容器加入到该网络，使网络中到容器可以通过容器名进行互相访问

docker network ls 查看当前docker 网络,docker 安装后有三种默认网络，分别为bridge,host,none
none docker 启动时使用--network=none指定，容器内只能使用loopback网络设备，不会有其他网络资源
host docker 启动时使用--network=host指定，容器使用当前宿主机的网络，容器内可以更改控制宿主机的网络
bridge docker 启动时使用--network=bridge指定，容器使用桥接的网络。

docker network create -d bridge my-bridge 创建一个类型为bridge，名为my-bridge的网络

docker run –name mynginx –network my-bridge -p 8080:80 -d nginx:latest 指定新启动的容器mynginx的网络：网络名为my-bridge的网络

docker network connect my-bridage test 手动将已运行中的容器test加入到网络my-bridge中

docker network rm [network id] 删除自定义的网络

docker compose

Compose项目是Docker官方的开源项目，负责实现对Docker容器集群的快速编排。通过使用YMAL文件配置可以定义和运行多容器的docker应用工具。然后通过一个命令，你能使用配置文件创建和运行所有的服务。
